PROFESSIONAL CALCULATOR APPLICATION
Python Developer Internship - Task 8
========================================

APPLICATION OVERVIEW
-------------------
This is a robust command line calculator application designed to perform 
mathematical operations while maintaining program stability through 
comprehensive exception handling. The application never crashes due to 
bad input or runtime errors.

SUPPORTED OPERATIONS
-------------------
1. Addition (+)
2. Subtraction (-)
3. Multiplication (*)
4. Division (/)
5. Power (^)
6. Modulus (| |)
7. Percentage (%)
8. View History
9. Clear History
10. Exit

EXCEPTION HANDLING STRATEGY
---------------------------

The application implements a multi layered exception handling approach 
to ensure maximum stability:

1. INPUT VALIDATION LAYER
   When users enter numbers, the application uses try except blocks to 
   catch ValueError exceptions. If a user enters letters, symbols, or 
   leaves the input empty, the program catches the error, displays a 
   friendly message, and retries the input without crashing. This 
   creates a seamless retry mechanism that keeps the application running.

2. OPERATION SELECTION VALIDATION
   When users choose menu options, the application validates that the 
   input is within the valid range of 1 to 10. Any invalid selection 
   triggers an error message and prompts the user to try again. The 
   program never exits due to invalid menu choices.

3. MATHEMATICAL OPERATION HANDLING
   Each mathematical operation is wrapped in try except blocks that 
   specifically handle:
   
   a) ZeroDivisionError
      When users attempt to divide by zero or perform modulus with 
      zero, the application catches this error and displays a clear 
      message explaining that division by zero is not allowed. The 
      program continues to accept new calculations.
   
   b) OverflowError
      When calculations produce numbers too large to handle, the 
      application catches this and informs the user without crashing.
   
   c) General Exceptions
      Any unexpected calculation errors are caught by a general 
      exception handler that displays an error message and allows 
      the user to continue.

4. KEYBOARD INTERRUPT HANDLING
   The application gracefully handles Ctrl+C keyboard interrupts. 
   Instead of showing a raw traceback error, it displays a friendly 
   message and exits cleanly with a summary of calculations performed.

5. GLOBAL EXCEPTION HANDLING
   The main application loop includes a try except block that catches 
   any unexpected exceptions that might bubble up from lower levels. 
   This ensures that even unanticipated errors do not crash the entire 
   program.

KEY EXCEPTION HANDLING FEATURES
------------------------------

RETRY MECHANISM
Instead of exiting on invalid input, the application repeatedly prompts 
the user until valid input is received. This is implemented using while 
loops that continue until valid data is entered.

MEANINGFUL ERROR MESSAGES
All error messages are user friendly and provide clear guidance on how 
to correct the mistake. Technical error details are hidden from users 
while maintaining professional communication.

NO TRACEBACKS
The application never displays Python traceback errors to users. All 
errors are caught, formatted, and presented in a clean professional 
manner.

CALCULATION CONTINUITY
When errors occur during calculations, the application displays the 
error but does not lose the calculation history or require restart. 
Users can continue with new calculations immediately.

HISTORY PERSISTENCE
The calculation history remains intact even when errors occur. Only 
failed calculations are excluded from history while successful ones 
are properly recorded.

ERROR CATEGORIES HANDLED
------------------------

1. Value Errors
   - Non numeric input
   - Empty input
   - Invalid number formats

2. Zero Division Errors
   - Division by zero
   - Modulus with zero

3. Keyboard Interrupt
   - User pressing Ctrl+C

4. General Exceptions
   - Unexpected calculation errors
   - System related issues
